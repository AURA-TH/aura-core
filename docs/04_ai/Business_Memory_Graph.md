# AURA Business Memory Graph

Version: 0.1
Project: AURA Intelligence Platform
Core Concept: Business Memory Graph
Status: Draft

---

## 1. Purpose

Business Memory Graph คือระบบความจำของธุรกิจ

หน้าที่ของ Business Memory Graph คือทำให้ AURA เข้าใจความสัมพันธ์ระหว่างข้อมูลทั้งหมดในธุรกิจ ไม่ใช่แค่เก็บข้อมูลเป็นตารางแยก ๆ

AURA ต้องรู้ว่า:

* ลูกค้าคนนี้คือใคร
* เคยซื้ออะไร
* สนใจสินค้าอะไร
* เคยคุยเรื่องอะไร
* มีปัญหาอะไร
* ชอบโปรโมชันแบบไหน
* ควรติดตามเมื่อไร
* มีโอกาสซื้อซ้ำหรือไม่

เป้าหมายคือทำให้ AI ทำงานเหมือนพนักงานที่รู้จักธุรกิจและลูกค้ามานานหลายปี

---

## 2. Why Memory Graph Matters

AI Model อย่าง GPT, Claude หรือ Gemini อาจเก่งมาก

แต่สิ่งที่ทำให้ AURA แตกต่างไม่ใช่โมเดล

สิ่งที่ทำให้ AURA แตกต่างคือ “ความจำของธุรกิจ”

คู่แข่งอาจใช้ AI Model ตัวเดียวกับเราได้
แต่คู่แข่งไม่มีข้อมูล ความสัมพันธ์ และประวัติของธุรกิจที่ AURA สะสมไว้

นี่คือ Moat สำคัญของ AURA

---

## 3. Core Idea

ข้อมูลธุรกิจไม่ได้อยู่แยกกัน

ทุกอย่างเชื่อมโยงกัน

```text
Customer
↓
ซื้อสินค้า
↓
Product
↓
อยู่ใน Brand
↓
อยู่ใน Category
↓
เกี่ยวข้องกับ Promotion
↓
สร้าง Revenue
↓
ส่งผลต่อ Business Insight
```

Business Memory Graph ต้องทำให้ AI เห็นความสัมพันธ์เหล่านี้

## 4. Main Nodes

Business Memory Graph จะประกอบด้วย Node หลักดังนี้

### 4.1 Business

ข้อมูลธุรกิจ

* Business ID
* ชื่อร้าน
* ประเภทธุรกิจ
* ช่องทางขาย
* นโยบายร้าน
* วิธีจัดส่ง
* วิธีชำระเงิน
* โทนการสื่อสาร
* เป้าหมายธุรกิจ

### 4.2 Customer

ข้อมูลลูกค้า

* Customer ID
* ชื่อ
* ช่องทางติดต่อ
* จังหวัด
* ประวัติการซื้อ
* ความสนใจ
* ปัญหาที่เคยเจอ
* พฤติกรรมการซื้อ
* โอกาสซื้อซ้ำ
* ระดับความสำคัญ

### 4.3 Product

ข้อมูลสินค้า

* Product ID
* ชื่อสินค้า
* ราคา
* หมวดหมู่
* แบรนด์
* จุดเด่น
* สต็อก
* ต้นทุน
* กำไร
* สินค้าที่เกี่ยวข้อง

### 4.4 Conversation

ข้อมูลบทสนทนา

* Conversation ID
* ลูกค้าที่เกี่ยวข้อง
* ช่องทางที่คุย
* คำถามหลัก
* สินค้าที่สนใจ
* สถานะการขาย
* ผลลัพธ์
* สรุปโดย AI

### 4.5 Order

ข้อมูลออเดอร์

* Order ID
* ลูกค้า
* สินค้า
* ยอดขาย
* กำไร
* สถานะจัดส่ง
* ช่องทางการชำระเงิน
* วันที่ซื้อ

### 4.6 Task

งานที่ต้องติดตาม

* Task ID
* ลูกค้าที่เกี่ยวข้อง
* งานที่ต้องทำ
* วันติดตาม
* ผู้รับผิดชอบ
* สถานะ
* ความสำคัญ

### 4.7 Insight

ข้อมูลวิเคราะห์

* Insight ID
* หัวข้อ
* สาเหตุ
* ข้อมูลที่ใช้
* คำแนะนำ
* ผลกระทบ
* Action ที่ควรทำ

## 5. Main Relationships

Graph ไม่ได้สำคัญแค่ Node
แต่สำคัญที่ความสัมพันธ์

ตัวอย่าง Relationship:

```text
Customer → asked_about → Product
Customer → bought → Product
Customer → complained_about → Order
Customer → prefers → Brand
Customer → responded_to → Promotion
Product → belongs_to → Category
Product → related_to → Product
Order → contains → Product
Conversation → mentions → Product
Conversation → created → Task
Insight → recommends → Action
```

ความสัมพันธ์เหล่านี้จะทำให้ AI เข้าใจบริบทได้ลึกกว่าการค้นหาข้อมูลธรรมดา

## 6. Example Memory

ตัวอย่างข้อมูลที่ AURA ควรจำได้:

```text
ลูกค้า: คุณสมชาย

ความสัมพันธ์:
- เคยซื้อ Royal Canin Kitten 2 ครั้ง
- มีแมว 3 ตัว
- ชอบ Flash Express
- เคยบ่นเรื่องค่าส่งแพง
- มักทักมาตอน 20:00 - 22:00
- มีโอกาสซื้อซ้ำทุก 30 วัน
- สนใจโปรโมชันแบบลดราคา มากกว่าของแถม
```

เมื่อคุณสมชายทักมาใหม่ AI ต้องตอบโดยใช้ข้อมูลเหล่านี้ได้ทันที

## 7. Use Case 1: AI ตอบลูกค้า

ลูกค้าถาม:

“อาหารแมวตัวเดิมยังมีไหม”

AI ต้องรู้ว่า “ตัวเดิม” คือสินค้าที่ลูกค้าเคยซื้อ

AI ต้องตอบได้ว่า:

* ลูกค้าเคยซื้ออะไร
* สินค้านั้นยังมีสต็อกไหม
* ราคาเท่าไร
* ควรแนะนำซื้อซ้ำหรือไม่
* มีโปรโมชันที่เกี่ยวข้องไหม

---

## 8. Use Case 2: AI Follow-up

ระบบพบว่า:

ลูกค้าเคยซื้ออาหารแมว 10 กิโลกรัมเมื่อ 28 วันก่อน

AI ควรสร้าง Task ว่า:

```text
ควรติดตามลูกค้าคนนี้ เพราะมีโอกาสซื้อซ้ำภายใน 2-3 วันนี้
```

AI สามารถร่างข้อความให้เจ้าของร้านอนุมัติ เช่น:

```text
สวัสดีค่ะคุณลูกค้า อาหารน้องแมวใกล้หมดหรือยังคะ ตอนนี้สินค้าที่เคยสั่งยังมีพร้อมส่งนะคะ
```

## 9. Use Case 3: AI Business Insight

เจ้าของร้านถาม:

“ทำไมยอดขายเดือนนี้ตก”

AI ต้องย้อนดูข้อมูลจาก Graph เช่น:

* สินค้าขายดีหมดสต็อก
* ลูกค้าเก่าซื้อซ้ำน้อยลง
* โปรโมชันเดิมไม่ได้ผล
* ลูกค้า VIP หายไป
* คู่แข่งลดราคา
* แอดมินตอบช้าลง

AI ต้องตอบพร้อมหลักฐาน ไม่ใช่เดา

## 10. Memory Types

Business Memory Graph ควรมี Memory หลายระดับ

### 10.1 Short-term Memory

ข้อมูลระหว่างบทสนทนาปัจจุบัน

ใช้เพื่อเข้าใจว่าลูกค้ากำลังพูดถึงอะไร

### 10.2 Long-term Memory

ข้อมูลสะสมระยะยาว เช่น ประวัติลูกค้า ประวัติการซื้อ และพฤติกรรมการซื้อซ้ำ

### 10.3 Business Rule Memory

ข้อมูลกฎของร้าน เช่น นโยบายส่งของ เงื่อนไขส่วนลด และเวลาทำการ

### 10.4 Decision Memory

ข้อมูลการตัดสินใจในอดีต เช่น เคยให้ส่วนลดลูกค้าคนนี้ หรือเคยมีข้อยกเว้นพิเศษ

### 10.5 Insight Memory

ข้อมูลวิเคราะห์ที่เคยสรุป เช่น สาเหตุยอดตก หรือสินค้าไหนควรดันขาย

## 11. Privacy and Security

Business Memory Graph จะเก็บข้อมูลสำคัญของธุรกิจและลูกค้า

ดังนั้นต้องมีหลักการด้านความปลอดภัย:

* แยกข้อมูลแต่ละธุรกิจออกจากกัน
* ห้าม AI เปิดเผยข้อมูลลูกค้าคนอื่น
* ต้องมีสิทธิ์การเข้าถึง
* ต้องมี Audit Log
* ข้อมูลสำคัญต้องถูกป้องกัน
* เจ้าของธุรกิจต้องควบคุมข้อมูลของตัวเองได้
* การลบข้อมูลต้องมีขั้นตอนยืนยัน

## 15. Final Principle

AURA ต้องไม่จำแค่ข้อความ

AURA ต้องจำความสัมพันธ์

AURA ต้องไม่รู้แค่ว่าลูกค้าถามอะไร

AURA ต้องรู้ว่าลูกค้าคนนี้เป็นใคร สำคัญแค่ไหน มีแนวโน้มจะซื้ออะไร และควรทำอะไรต่อ

Business Memory Graph คือหัวใจที่ทำให้ AURA กลายเป็น Business Brain อย่างแท้จริง
